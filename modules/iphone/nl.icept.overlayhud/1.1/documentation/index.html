overlayhud Module 1.1

Description

OverlayHUD is a generic component to build HUD's. You can use this for example to show notifications of completed tasks, a blocking notification of a busy task or a login box. The userinput to underlaying views is blocked to prevent bypassing of the HUD. But in a tabView you are still free to switch to a different tab.

2 kinds of HUDs are included. A generic empty hud and a loading hud with a label, activity indicator and the posibility to show a image or image sequence. The loading HUD is similar to MBProgressHUD wich is a native component with a similar functionality.

Every hud is customizable with the posibility to add additional subviews and elements.

Accessing the overlayhud Module

To access this module from JavaScript, you would do the following:

var overlayhudModule = require("nl.icept.overlayhud");

The overlayhudModule variable is a reference to the Module object.

Don't forget to list the module in tiapp.xml!

Reference

createEmptyHUD(options)

Create a empty HUD. Options is a optional object with the following properties and default values:

{ zIndex: 999,

backgroundColor: "#000",
borderRadius: 10,
opacity: 0.8,

height: 'auto',
width: 'auto',
top: null,
bottom: null,
left: null,
right: null

}

createEmptyHUDOnWindow(window, options)

Create a empty HUD on the specified window. Use this when you use the Single Context Model.

createLoadingHUD(options)

Create a loading HUD. Options is a optional object with the following properties and default values:

{ zIndex: 999,

backgroundColor: "#000",
borderRadius: 10,
opacity: 0.8,

color: '#fff',
fontSize: 20,
fontWeight: 'bold',

message: "Loading...",

activityIndicator: {
    style: Titanium.UI.iPhone.ActivityIndicatorStyle.BIG,
    hidden: false,
    top: null,
    bottom: null,
    left: null,
    right: null
},

image: {
    image: null,
    images: null,
    opacity: 1.0,
    repeatCount: 1,
    hidden: false,
    height: 'auto',
    width: 'auto',
    top: null,
    bottom: null,
    left: null,
    right: null
},

label: {
    hidden: false,
    height: 'auto',
    width: 'auto',
    top: null,
    bottom: 20,
    left: null,
    right: null
},

overlay: {
    height: 150,
    width: 150,
    top: null,
    bottom: null,
    left: null,
    right: null
}

}

createLoadingHUDOnWindow(window, options)

Create a loading HUD on the specified window. Use this when you use the Single Context Model.

.show()

Show the HUD. Its attached to the current window.

.hide()

Hide the HUD.

.add(element)

Add a new element or subview to the HUD.

.remove(element)

Remove a element or subview from the HUD.

Usage

var loadingOverlay = overlayhudModule.createLoadingHUD({}); loadingOverlay.show();

setTimeout(function () { loadingOverlay.hide({}); }, 3000);

For a complete overview of the options, a example project is included.
